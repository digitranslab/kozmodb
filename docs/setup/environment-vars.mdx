Most of the KozmoDB functionality can be modified by extending the default configuration, but some of the configuration options
can be added as environment variables on the server where KozmoDB is deployed.

## KozmoDB Authentication

KozmoDB does not require authentication by default. If you want to enable authentication, you can set the `KOZMODB_USERNAME` and `KOZMODB_PASSWORD` environment variables.

### Example

<CodeGroup>
    ```bash Docker
    docker run --name kozmodb_container -e KOZMODB_USERNAME='kozmodb_user' -e KOZMODB_PASSWORD='kozmodb_password' -e KOZMODB_APIS=http,mysql -p 47334:47334 -p 47335:47335 digitranslab/kozmodb
    ```

    ```bash Shell
    export KOZMODB_USERNAME='kozmodb_user'
    export KOZMODB_PASSWORD='kozmodb_password'
    ```
</CodeGroup>

## KozmoDB Configuration File

In order to start KozmoDB with a [custom configuration file](/setup/custom-config), the `KOZMODB_CONFIG_PATH` environment variable should store the file path.

### Example

<CodeGroup>
    ```bash Docker
    docker run --name kozmodb_container -e KOZMODB_CONFIG_PATH=/Users/username/path/config.json -e KOZMODB_APIS=http,mysql -p 47334:47334 -p 47335:47335 digitranslab/kozmodb
    ```

    ```bash Shell
    export KOZMODB_CONFIG_PATH=/Users/username/path/config.json
    ```
</CodeGroup>

## KozmoDB Storage

By default, KozmoDB stores the configuration files by determining appropriate platform-specific directories, e.g. a "user data dir":

* On Linux `~/.local/share/kozmodb/var`
* On MacOS `~/Library/Application Support/kozmodb/var`
* On Windows `C:\Documents and Settings\<User>\Application Data\Local Settings\<AppAuthor>\kozmodb\var`

In the `KOZMODB_STORAGE_DIR` location, KozmoDB stores users' data, models and uploaded data files, the static assets for the frontend application and the
`sqlite.db` file. 
You can change the default storage location using `KOZMODB_STORAGE_DIR` variable.

### Example

<CodeGroup>
    ```bash Docker
    docker run --name kozmodb_container -e KOZMODB_STORAGE_DIR='~/home/kozmodb/var' -e KOZMODB_APIS=http,mysql -p 47334:47334 -p 47335:47335 digitranslab/kozmodb
    ```

    ```bash Shell
    export KOZMODB_STORAGE_DIR='~/home/kozmodb/var'
    ```
</CodeGroup>

## KozmoDB Configuration Storage

KozmoDB uses `sqlite` database by default to store the required configuration as models, projects, files metadata etc.
The full list of the above schemas can be found [here](https://github.com/digitranslab/kozmodb/blob/main/kozmodb/interfaces/storage/db.py#L69). You can change the
default storage option and use different database by adding the new connection string using `KOZMODB_DB_CON` variable.

### Example

<CodeGroup>
    ```bash Docker
    docker run --name kozmodb_container -e KOZMODB_DB_CON='postgresql://user:secret@localhost' -e KOZMODB_APIS=http,mysql -p 47334:47334 -p 47335:47335 digitranslab/kozmodb
    ```
    
    ```bash Shell
    export KOZMODB_DB_CON='postgresql://user:secret@localhost'
    ```
</CodeGroup>

## KozmoDB APIs

The `KOZMODB_APIS` environment variable lets users define which APIs to start. Learn more about the [available APIs here](/setup/kozmodb-apis).

### Example

<CodeGroup>
    ```bash Docker
    docker run --name kozmodb_container -e KOZMODB_APIS=http,mysql -p 47334:47334 -p 47335:47335 digitranslab/kozmodb
    ```

    ```bash Shell
    export KOZMODB_APIS='http,mysql'
    ```
</CodeGroup>

## KozmoDB Logs

This environment variable defines the level of logging generated by KozmoDB. You can choose one of the values [defined here](https://docs.python.org/3/library/logging.html#logging-levels). The `INFO` level is used by default.

### Example

<CodeGroup>
    ```bash Docker
    docker run --name kozmodb_container -e KOZMODB_LOG_LEVEL='DEBUG' -e KOZMODB_APIS=http,mysql -p 47334:47334 -p 47335:47335 digitranslab/kozmodb
    ```

    ```bash Shell
    export KOZMODB_LOG_LEVEL='DEBUG'
    ```
</CodeGroup>

## KozmoDB Default Project

By default, KozmoDB creates a project named `kozmodb` where all the models and other objects are stored. You can change the default project name by setting the `KOZMODB_DEFAULT_PROJECT` environment variable.

If this environment variable is set or modified after KozmoDB has started, the default project will be **renamed** accordingly upon restart. To start using the new default project, a `USE` statement will also need to be executed.

### Example

<CodeGroup>
    ```bash Docker
    docker run --name kozmodb_container -e KOZMODB_DEFAULT_PROJECT='my_project' -e KOZMODB_APIS=http,mysql -p 47334:47334 -p 47335:47335 digitranslab/kozmodb
    ```

    ```bash Shell
    export KOZMODB_DEFAULT_PROJECT='my_project'
    ```
</CodeGroup>

## KozmoDB's PID File

When running KozmoDB via [Docker](/setup/self-hosted/docker) or [Docker Extension](/setup/self-hosted/docker-desktop), the PID file is not used by default. Users can opt for enabling the PID file by defining the `USE_PIDFILE` environment variable.

If used, the PID file is stored in the temp directory (`$TMPDIR` on MacOS and Linux, `%TEMP%` on Windows) under the `kozmodb` folder.

### Example

<CodeGroup>
    ```bash Docker
    docker run --name kozmodb_container -e USE_PIDFILE=1 -e KOZMODB_APIS=http,mysql -p 47334:47334 -p 47335:47335 digitranslab/kozmodb
    ```

    ```bash Shell
    export USE_PIDFILE=1
    ```
</CodeGroup>

## KozmoDB GUI

In order to disable automatic GUI updates, the `KOZMODB_NO_STUDIO` environment variable should be set to `true`.

### Example

<CodeGroup>
    ```bash Docker
    docker run --name kozmodb_container -e KOZMODB_NO_STUDIO=true -e KOZMODB_APIS=http,mysql -p 47334:47334 -p 47335:47335 digitranslab/kozmodb
    ```

    ```bash Shell
    export KOZMODB_NO_STUDIO=true
    ```
</CodeGroup>
